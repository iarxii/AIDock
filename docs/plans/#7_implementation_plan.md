# Refactoring Hardcoded Secrets and Configurations

This plan outlines the steps to remove hardcoded configuration values and secrets from the codebase, replacing them with a unified `route_map.json` generated by the `AI_Codex` deployment pipeline. This ensures seamless integration between `AIDock` and `AI_Codex` without needing complex Docker `.env` injection logic.

## User Review Required

> [!NOTE]
> The `CLOUD_BE` URL will now be sourced from `route_map.json` within the `AIDock/client/src/` directory. When `deploy_production.bat` runs in the `AI_Codex` repository, it will automatically update this file. A Docker rebuild (`aidock reload`) will still be required if the `route_map.json` changes, as Vite bundles imported JSON files.

## Proposed Changes

### Integration with AI_Codex Pipeline

#### [MODIFY] [deploy_production.bat](file:///c:/AppDev/My_Linkdin/projects/iarxii/AI_Codex/deploy_production.bat)
- Update the deployment script in the `AI_Codex` repository to generate a copy of `route_map.json` at `..\AIDock\client\src\route_map.json`. This ensures that whenever `AI_Codex` is deployed, `AIDock` immediately receives the correct, up-to-date Cloud Run routing variables.

### Frontend Configuration (Vite)

#### [MODIFY] [App.tsx](file:///c:/AppDev/My_Linkdin/projects/iarxii/AIDock/client/src/App.tsx)
- Replace hardcoded `activeModel` initialization (`'Gemma4 E4B'`) with `'Detecting...'`. The `useEffect` block immediately fetches the active model from the `/info` backend endpoint, making the hardcoded value obsolete.
- Import `route_map.json` at the top of the file: `import routeMap from './route_map.json';`.
- Replace `const CLOUD_BE = 'https://...';` with `const CLOUD_BE = routeMap.backend_url;`.
- **Security Check**: Hide the Cloud Run backend endpoint URL from the Settings Diagnostics panel. Update the Active Chat Endpoint display to show a generic obscured path (e.g., `https://[HIDDEN-FOR-SECURITY]/api/chat`) when `isCloudMode()` is active.
